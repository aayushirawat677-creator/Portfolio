Not an advice tool — a possibility engine. Ask any question, get both honest futures, explore each timeline card by card. Never told what to do.

The problem with AI decision tools.

Every AI tool that touches decisions falls into the same trap: it gives advice. "You should do X." "Option B is better for you." "I recommend..."

That's the wrong model. The user knows their life better than any AI. What they need is clarity — a way to see both sides of a choice without someone else deciding for them.

What If? is built on one principle: show what could happen, never what to do.

The user types a question or statement. A 3-agent Azure AI Foundry pipeline classifies the decision, gathers targeted context, and generates two honest what-if timelines. No recommendation. No judgment. Just possibilities.

Who it's for.

Life decision-makers

People facing real decisions — career moves, housing, relationships, daily choices — who want to think clearly without being told what to do.

Overthinkers

People who get stuck in their own heads and need to externalize both paths to move forward.

The curious

People who want to explore "what if" scenarios not because they're paralyzed, but because they're genuinely curious about both futures.

Query Modes

Three ways a decision can be framed.

The Baseline Classifier (Agent 1) identifies which mode the user's input belongs to — and this determines everything downstream: path labels, polarity, output shape, and what context to gather.

// mode 01

binary_decision

A yes/no gate fits the question.
Example: "Should I quit my job to start a company?"

Paths: yes / no
Polarity: assigned per path
// mode 02

option_compare

User presents two specific options.
Example: "MacBook or Windows laptop?"

Paths: option_a / option_b
Polarity: none — purely factual
// mode 03

statement_assessment

User states a fact, implying an implicit decision.
Example: "I am 100kg."

Paths: yes (act) / no (don't)
Polarity: agent-assigned
AI Architecture

A 3-agent pipeline on Azure AI Foundry.

Natural-language decision becomes two honest futures, card by card.

All agents run on gpt-4.1-mini via Azure AI Foundry. The pipeline is orchestrated from the Next.js API layer — agents are called sequentially, with context passed between each stage. No agent can recommend or advise — the user's decision is sacred.

// the non-negotiable rule
output = possibilities, never advice
// the job of each agent
success = user_makes_own_decision ? true : false
Agent 1

Baseline Classifier — the decision decoder.

Turns a messy question into a structured decision that sets up the entire downstream pipeline.

// input

User question

Natural language. Messy. Emotional. Real.

// output

Structured JSON

query_mode, paths, polarity, required_context, safety_flag

// key rule

Agent decides polarity

Not the user. "Skip my workout?" → yes=negative, no=positive.

Why this matters: The Classifier pre-defines exactly what context Agent 2 needs to collect for each path. This prevents irrelevant questions and keeps the experience tight.

Agent 2

Context Gatherer — one question at a time.

Fills in the context checklist from Agent 1 without interrogating the user. Maximum 3 questions.

// game feel

One Q per turn

Not a form. Conversational. Fast.

// both sides

Both paths matter

Collects context for yes AND no, not just the likely path.

// max 3

Hard stop

After 3 questions, generate regardless. User came for answers.

Key design: Alternates between collecting context for path A and path B. The user's initial answer (yes/no) doesn't skip context for the other path — both futures need grounding.

Agent 3

What-If Generator — two honest futures.

Writes vivid, specific, causally connected timelines for each path. No utopia. No doom.

// second person

Specific to you

"You start the gym and feel better" is failure. "By week 3 your lower back pain eases" is the target.

// realism

Every path has friction

No perfect outcomes. No doom scenarios. Just real obstacles.

// causality

Checkpoint logic

Checkpoint 2 follows checkpoint 1; checkpoint 3 from checkpoint 2.

Safety & Guardrails

Three layers of protection.

The first one never reaches an agent. Code is deterministic — agents can be jailbroken.

Layer 1 — Code Guardrails (instant, <1ms)

Design principle: Safety checks run in code, not agents. Pattern matching is faster and more reliable than LLM judgment for hard stops.

// hard stop

Self-harm / Suicide

💙 988 Lifeline + Crisis Text Line

// hard stop

Harm to Others

🚨 Emergency services + legal warning

// hard stop

Hard Drugs

⚠️ SAMHSA 1-800-662-4357

// hard stop

Toxic Substances

🏥 Poison Control 1-800-222-1222

// hard stop

Medical Decisions

Doctor required (no AI advice)

// hard stop

Illegal Activities

Legal consequences only, no encouragement

Layer 2 — Age Gate (Alcohol & Tobacco)

// alcohol

Age 21+

Under 21 → polite redirect with humor ("Your liver will thank you later — raise a glass of OJ 🍊")

// smoking

Age 18+

Under 18 → health redirect. Legal age confirmed → disclaimer popup → proceed with honest what-ifs

Layer 3 — Agent Prompt Rules

✓

Never advise

Never say "you should", "you must", "I recommend", "the best option"

✓

Always include friction

Every scenario must have at least one realistic obstacle

✓

Polarity as trajectory

Describes tone direction, never judgment of the person

✓

Option compare is neutral

Never uses polarity — purely factual comparison

Evaluation Framework

Seven dimensions of quality.

A dedicated evaluation agent that scores pipeline output across critical dimensions.

1

STRUCTURAL

Does output match the required JSON schema exactly?

2

ACCURACY

Correct query_mode, time_impact, polarity_applies?

3

QUALITY

Specific, realistic, non-generic scenarios?

4

SAFETY

Correct handling of dangerous inputs?

5

NO_ADVICE

Avoids "should", "recommend", "best option"?

6

FRICTION

At least one realistic obstacle in every scenario?

7

DIVERSITY

Are the two paths genuinely different from each other?

Returns: scores 0-10 per dimension, overall score, passed: true/false, issues list, one-sentence verdict.

Test coverage: 10 test cases covering binary decisions, option comparisons, and statement assessments across real-world decision scenarios.

Product principles.

Five rules the entire system is built around.

✓

Show possibilities, never advice

The user decides, always. The AI's job is clarity, not direction.

✓

Honest over optimistic

Every path includes realistic friction. No path is sold as the obvious right choice.

✓

Minimum viable questions

Max 3 follow-ups before generating. The user came for answers, not a survey.

✓

Safety first, always in code

Guardrails run before any agent call. Deterministic checks for hard stops.

✓

Specific over generic

Timelines must reflect the user's actual situation. Generic outcomes are failure.
